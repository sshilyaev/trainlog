## Данные и аналитика

- **Фото прогресса** — привязка фото к дате/замеру (Firebase Storage или локально).
- **Процент жира / мышечной массы** — опциональные поля в замере или отдельная метрика; при желании — расчёт по формулам.
- **Сравнение двух замеров** — экран «было → стало» по выбранным датам (числа + опционально мини-график).
- **Тренды** — подписи под графиками: «−2 кг за месяц», «стабильно» и т.д.
- **Недельная/месячная сводка** — карточка на дашборде: средний вес за неделю, изменение относительно прошлой недели.

## UX для тренера

- **Групповые действия** — массовое создание абонементов (один шаблон на нескольких подопечных).
- **Заметки к визиту** — поле «заметка» у визита (что делали, самочувствие) для истории.
- **Фильтр посещений** — по статусу (долг / оплачено / отменено), по периоду.
- **Быстрый ввод визита** — сегодня одной кнопкой «Отметить приход» без выбора даты.
- **Статистика по клиенту** — сколько визитов за месяц/квартал, сколько абонементов использовано за всё время.
- **Добавление посещения по тапу на день в календаре** — разовое или с выбором абонемента. ✅ Реализовано.

---

## UX для подопечного

- **Виджет (iOS)** — вес «сейчас» или последний замер на домашнем экране.
- **Быстрый замер** — экран/шторка только вес (+ опционально 1–2 поля), без полной формы.
- **Цели с напоминанием** — «Напомнить за N дней до целевой даты».
- **Несколько единиц** — переключатель кг/фунты, см/дюймы (если будет аудитория не из метрики).

---

## Связь и безопасность

- **Cloud Functions для токенов** — генерация/валидация/очистка на сервере (если усиливать безопасность).
- **Приглашение по ссылке** — вместо только кода: ссылка вида `trainlog://connect?token=XXX`, открывает приложение и подставляет токен.
- **Подтверждение отзыва доступа** — тренер отвязывает подопечного → подопечному пуш «Тренер X отменил связь» (при появлении серверной рассылки).

---

## Общее и полировка

- **Onboarding** — 1–2 экрана при первом запуске: что такое профили, замеры, как подключить тренера.
- **Подсказки (tooltips)** — первый заход на экран «Абонементы» / «Посещения» и т.д.; возможность сбросить прохождение и пройти снова. ✅ Реализовано.
- **Локализация** — вынос строк в Localizable.strings, при желании английский.
- **Accessibility** — VoiceOver, увеличенный текст, контраст (аудит по PREPRODUCTION_STANDARDS).

---

## Дальнейшее развитие продукта

- **Несколько тренеров у одного подопечного** — явная поддержка в UI «несколько тренеров», разделение посещений/абонементов по тренерам.
- **Платежи** — оплата абонемента через IAP или внешний платёж (отдельный большой пласт).
- **Резервное копирование / перенос** — экспорт всех своих данных в один файл и восстановление на новом устройстве или после переустановки.
