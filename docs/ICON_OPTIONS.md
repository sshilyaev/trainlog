## Вариант 1: Один стиль SF Symbols (filled vs outline)

**Идея:** Везде придерживаться одного визуального веса — либо в основном **filled** (заливка), либо **outline** (контур).

- **Сейчас:** микс — есть `person.circle.fill`, `person.3.fill`, но `target`, `list.bullet`, `number` без заливки.
- **Сделать:**  
  - Табы и главные действия: только **filled** (например `person.circle.fill`, `person.3.fill`, `target`, заменить `list.bullet` на `list.bullet` в табе или подобрать filled-аналог по смыслу).  
  - Вторичные действия и пустые состояния: либо тоже filled, либо единый outline для «спокойных» блоков.

**Плюсы:** Быстро, без новых ресурсов, интерфейс визуально упорядочен.  
**Минусы:** Не все символы есть в filled-варианте, часть останется outline.

## Вариант 2: Семантика под TrainLog (другое значение тех же SF Symbols)

**Идея:** Не менять «стиль», а заменить часть иконок на более подходящие по смыслу замеров/целей/тренера.

Примеры замен:

| Сейчас | Вариант замены | Зачем |
|--------|----------------|-------|
| `number` (код) | `key` или `link` | Код = ключ/связь |
| `list.bullet` (замеры) | `ruler` или `ruler.fill` | Замеры = линейка |
| `target` (цели) | оставить или `scope` | Цель = прицел |
| `figure.arms.open` (splash) | `figure.run`, `heart.fill` или оставить | Фитнес/тело |
| `rectangle.2.swap` (смена профиля) | `arrow.left.arrow.right` или `person.2` | Смена аккаунта |
| Пусто: замеры `list.bullet.clipboard` | `ruler` или `chart.bar.doc.horizontal` | Ближе к замерам |

Остальное (person.*, plus.circle.fill, chart.xyaxis.line, share, copy, trash, error) можно оставить или точечно подобрать из SF Symbols.

**Плюсы:** Без кастомных картинок, но интерфейс «про приложение» сильнее.  
**Минусы:** Нужно пройтись по экранам и согласовать замены.

## Вариант 3: SF Symbols — одна «семья» (Human / Fitness)

**Идея:** Максимально использовать символы из категорий **Human** и **Fitness/Health**: `figure.*`, `heart.*`, `bolt.*`, `flame.*`, плюс нейтральные `chart.*`, `list.*`, `person.*`.

- Splash: оставить `figure.arms.open` или взять `figure.run`, `figure.strengthtraining.traditional`.
- Профиль/подопечные: оставить `person.*` / `person.3.*`.
- Цели: `target` или `flag.checkered`.
- Замеры: `ruler`, `ruler.fill` или `list.bullet`.
- Графики: оставить `chart.xyaxis.line`.

**Плюсы:** Ощущение «про тело/активность» при минимальных правках.  
**Минусы:** Не все экраны получится строго из одной семьи.

## Вариант 4: Свои иконки в Asset Catalog

**Идея:** Набор кастомных изображений (PDF/Vector или @1x @2x @3x) в Assets: логотип, «профиль», «подопечные», «цели», «замеры», «графики», «код», «добавить», «поделиться», «ошибка» и т.д.

- В коде вместо `Image(systemName:)` использовать `Image("имя_из_каталога")` и при необходимости задать рендер-режим (template vs original).
- Табы и списки: `Label(..., image: Image("icon_name"))`.

**Плюсы:** Уникальный вид, полный контроль над стилем.  
**Минусы:** Нужно нарисовать/подобрать и обновлять при изменении экранов.

## Что выбрать

| Вариант | Сложность | Время | Результат |
|---------|-----------|--------|-----------|
| 1. Один стиль (filled/outline) | Низкая | 30–60 мин | Более единый интерфейс |
| 2. Семантика под TrainLog | Низкая | ~1 ч | Иконки лучше по смыслу |
| 3. Семья Human/Fitness | Низкая | ~1 ч | Тема «тело/фитнес» |
| 4. Кастом в Assets | Высокая | от нескольких часов | Уникальный дизайн |

Можно комбинировать: например **1 + 2** (единый стиль SF + точечные замены по смыслу) даёт быстрый и понятный результат без кастомных ресурсов.

## Реализовано: вариант 1+2

**Семантические замены:**
- `number` → `key` (Подключить по коду, Добавить по коду)
- `list.bullet` / `list.bullet.clipboard` → `ruler.fill` (замеры)
- `rectangle.2.swap` → `arrow.left.arrow.right` (сменить профиль)

**Единый стиль (filled):**
- Пусто: подопечные `person.3` → `person.3.fill`
- Копировать: `checkmark` / `doc.on.doc` → `checkmark.circle.fill` / `doc.on.doc.fill`
- Поделиться: `square.and.arrow.up` → `square.and.arrow.up.fill`
- Ошибка: `exclamationmark.triangle` → `exclamationmark.triangle.fill`
- Удалить: `trash` → `trash.fill`

Остальное (person.circle.fill, person.3.fill, target, chart.xyaxis.line, plus.circle.fill, person.badge.key.fill, person.fill, chevron.right/left) оставлено без изменений.
