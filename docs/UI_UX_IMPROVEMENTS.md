# TrainLog — идеи по улучшению UI/UX

> Сохраняем «лёгкий взгляд»: без перегруза, мягкий акцент, понятные действия.

---

## 1. Пустые состояния (Empty states)

Сейчас: просто текст «Замеров пока нет», «Целей пока нет», footer у подопечных.

**Идея:** использовать `ContentUnavailableView` с иконкой и коротким текстом — так экран не выглядит «пустым», а подсказывает, что делать.

| Где | Сейчас | Вариант улучшения |
|-----|--------|-------------------|
| Замеры (подопечный) | «Замеров пока нет» | ContentUnavailableView: иконка списка, «Пока нет замеров», «Нажмите «Добавить замер»» |
| Цели (подопечный) | «Целей пока нет» | То же с иконкой цели и текстом про «Добавить цель» |
| Подопечные (тренер) | Footer под списком | При пустом списке — блок с иконкой и «Добавьте подопечного по коду или свой профиль» |
| Выбор профиля | Список + кнопка | При 0 профилях — ContentUnavailableView «Создайте первый профиль» с кнопкой |

## 2. Состояние загрузки

| Где | Проблема | Вариант |
|-----|----------|---------|
| Карточка клиента (ClientCardView) | Пока грузятся замеры/цели — пустой список | Показать ProgressView или скелетон, пока данные не пришли |
| Список подопечных | При refresh уже есть список | Оставить как есть (pull-to-refresh) |

---

## 3. Ошибки

| Где | Сейчас | Вариант |
|-----|--------|---------|
| Вход / Регистрация | Красный текст под полями | Оставить текст или заменить на alert при ошибке |
| Создать профиль | Секция с красным текстом + кнопка «Понятно» | Можно вынести в alert, чтобы форма не «разъезжалась» |
| Подключить по коду / Добавить по коду | Уже есть ContentUnavailableView / alert | Оставить как есть |

---

## 4. Единообразие

- **Главные кнопки:** везде `.borderedProminent` + `.tint(AppDesign.accent)` (Войти, Создать, Подтвердить код и т.д.) — в основном уже так.
- **Поля ввода:** везде `.roundedBorder` или один общий стиль — проверить Auth и CreateProfile.
- **Списки:** везде `.listStyle(.insetGrouped)` — уже используется.

---

## 5. Мелкие улучшения

- **Splash:** чуть больше отступов, опционально лёгкая анимация появления (например, opacity).
- **Табы:** подписи «Профиль», «Подопечные», «Замеры», «Графики» — уже понятные; при желании можно проверить иконки.
- **Кнопка «Выйти»:** уже `role: .destructive` — ок.
- **Копировать код:** уже есть обратная связь «Скопировано» — ок.

---

## 6. Что не трогать (пока)

- Навигация и структура экранов — логичные.
- Цвета и акцент — оставить как есть.
- Формы создания профиля / замера / цели — не перегружать.

## Что уже сделано (вне списка выше)

- Лоадер при загрузке профилей после входа.
- **Запоминание последнего выбранного профиля** — при перезаходе в приложение сразу открывается главный экран с ним (UserDefaults).
- **По умолчанию открывается вкладка «Профиль»** в нижнем меню (тренер и подопечный).
- **Поиск/фильтр в списке подопечных** — по имени или заметке.
- **Загрузка в карточке клиента** — параллельная загрузка замеров, целей, абонемента и визитов; при первом заходе на список подопечных — лоадер до загрузки.
- Цели в карточке клиента у тренера; в GoalRow отображается тип цели (Вес, Талия и т.д.).
- Отвязка подопечного у тренера (кнопка «Отвязать подопечного» в карточке).
- Удаление профиля из самого профиля (вкладка «Профиль» у тренера и подопечного — кнопка «Удалить профиль» с подтверждением).
- Графики: без зазоров, каждое значение по оси X; переход в конкретный график по явному NavigationLink.
- В списке подопечных: заметка рядом с именем, без подписи «Подопечный».
- Кнопки «Войти» и «Создать аккаунт» — стандартный системный стиль (без зелёного акцента).
- Кнопка «Назад» в «Добавить по коду» исправлена (возврат только с экрана кода).
- **Пустые состояния:** замеры и цели — ContentUnavailableView с увеличенными отступами; список подопечных — краткое описание «Добавьте подопечного по коду или свой профиль»; выбор профиля при 0 профилях — ContentUnavailableView «Создайте первый профиль» с кнопкой.
- **Splash:** увеличены отступы (56/80), анимация появления сохранена.
- **Ошибки сохранения/удаления** замеров и целей отображаются в alert в TraineeMainView.
