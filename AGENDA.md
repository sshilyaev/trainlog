## Обзор

iOS-приложение для отслеживания измерений тела с двумя типами профилей: **тренер** (работа с подопечными) и **подопечный** (личный дневник замеров). Один аккаунт — множество профилей разных типов.

---

## Профили

- Один аккаунт (Firebase Auth) → несколько профилей.
- Типы профиля: **тренер** и **подопечный**.
- У каждого тренерского профиля свой список клиентов.
- Подопечный профиль может быть:
  - **Собственным** — пользователь вводит свои замеры.
  - **Управляемым** — пользователь вводит замеры за другого человека (без отдельного аккаунта).
- **Связь «свой подопечный ↔ свой тренер»**: при наличии тренерского профиля можно добавить свой trainee-профиль в список клиентов через «Добавить мой профиль».

## Абонементы и посещения (реализовано)

Учёт занятий и абонементов между тренером и подопечным.

### Абонемент

- **Модель:** пакет из N занятий на пару тренер–подопечный. Поля: количество занятий, использовано, дата создания, опционально цена (₽), статус.
- **Статусы абонемента:** активен (`active`), завершён (`finished`), отменён (`cancelled`). Активен = есть остаток занятий и статус active.
- **Номер абонемента:** при создании присваивается отображаемый код «буква + цифра» (A1, A2, … A9, B1, …) — для удобства в списках и в посещениях.
- **Кто создаёт:** тренер в карточке подопечного → раздел «Абонементы» → «Новый абонемент» (количество занятий, опционально цена).

### Посещение (визит)

- **Кто ведёт:** только тренер. Подопечный видит посещения в режиме только просмотра.
- **Действия:** тренер в карточке подопечного → «Посещения» → «Отметить приход» (выбор даты) → создаётся визит и сразу отмечается как проведённый.
- **Статусы визита:** запланировано (`planned`), проведено (`done`), отменено (`cancelled`), не пришёл (`noShow`). В UI подпись «Пришёл» и подсветка «пришёл» в календаре не показываются — только «Не пришёл», «Запланировано», «Отменено» и состояние оплаты/долга.
- **Оплата визита:** не оплачено (`unpaid`), оплачено (`paid`), долг (`debt`).
  - При «Отметить приход»: если есть активный абонемент с остатком занятий — списывается одно занятие, визит помечается оплаченным и привязывается к этому абонементу (в карточке визита показывается номер абонемента, например «С аб. A1»).
  - Если активного абонемента нет или в нём 0 занятий — визит помечается как **долг**.
- **Погашение долга:** у визита со статусом «долг» тренер может вызвать контекстное действие «Списать с абонемента» — в меню перечислены **все** абонементы с остатком занятий; при выборе списывается одно занятие с выбранного абонемента и визит переводится в оплаченные с привязкой к нему.
- Вручную пометить визит «оплачено» без списания с абонемента в текущем UI нет (есть только списание с абонемента).
- **Отмена визита:** если визит уже создан, его можно отменить. Тогда визит помечается `cancelled`, оплата и привязка к абонементу снимаются, а если занятие было списано — оно возвращается обратно. Долг при отмене не выставляется.

### Расписание

- **Календарь занятий на будущее не реализован.** Есть только журнал фактов: дата посещения и статус. Тренер при «Отметить приход» выбирает дату (в том числе прошедшую). Идеи на будущее: календарь тренировок, напоминания о занятии.

## Цели

- Формат: **значение + дата**.
- Можно задать несколько целей на одну метрику.
- Цели сохраняются в истории.
- На графике: **горизонтальная линия** целевого значения.

## Подключение тренер ↔ подопечный

1. Подопечный генерирует временный токен (QR + код).
2. Тренер сканирует QR или вводит код.
3. Тренер выбирает свой тренерский профиль, к которому привязывается подопечный.
4. **Внутренняя связь**: «Добавить мой профиль» — выбор своего trainee-профиля и добавление в список клиентов тренерского профиля (если такой профиль есть).
